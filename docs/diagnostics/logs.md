# Diagnostics and logs

The Help menu contains local diagnostics tools for troubleshooting without
guessing what happened.

## Where to find it

Menu -> **Help**.

Always available:

- **Copy diagnostics summary** — opens a summary dialog you can copy into a bug
  report.
- **Save local diagnostics** — writes app logs and queued crash reports to your
  Documents folder.

When diagnostics are enabled in your app config, additional items appear:

- **Capture walkthrough screenshots** — automatically captures the main
  screens used by the help site.
- **Save logs to Documents** — writes app logs and crash reports locally.
- **Verbose logs** — toggles extra logging for the current run.
- **Upload crash logs** — sends queued crash reports to the configured crash
  upload location.
- **Upload app logs + screenshots** — sends the app log plus any screenshots
  captured by the diagnostics screenshot button.

## Screenshot button

When diagnostics are enabled, a small floating screenshot button appears over
the app. Dialogs that run in a separate Android window, such as catalog details
and game settings, include their own screenshot button inside the dialog so the
dialog can be captured too.

Screenshots are stored with diagnostics and included when you upload app logs.

## Privacy

Diagnostics are explicit user actions. The app does not upload logs or
screenshots automatically.
